# Класифікація номіналу монет євро

## Тема проєкту

**«Дослідження різних методів багатокласової класифікації для визначення номіналу монети євро на основі набору даних EURO coins»**.

Мета роботи — перетворити набір для детекції об'єктів на набір для класифікації, виділити числові ознаки з монет і порівняти десять класичних алгоритмів машинного навчання.

## Набір даних

Каталог `data/images/` містить 150 реальних фотографій приблизного розміру `720x960`. На одній фотографії може бути кілька монет. Каталог `data/labels/` містить YOLO-анотації у форматі:

```text
class_id center_x center_y width height
```

Координати нормалізовані в діапазоні від 0 до 1. Під час підготовки кожна рамка переводиться в пікселі за фактичним розміром фотографії, монета вирізається та масштабується до `64x64`. У результаті отримано 336 окремих кропів.

## Встановлення

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Проєкт використовує NumPy, Pillow, pandas, Matplotlib, seaborn, scikit-image, scikit-learn і Jupyter. TensorFlow та PyTorch не потрібні.

## Запуск

Відкрити ноутбук інтерактивно:

```bash
jupyter notebook notebooks/euro_coin_classification.ipynb
```

Або виконати його повністю з командного рядка:

```bash
jupyter nbconvert \
  --execute \
  --to notebook \
  --inplace notebooks/euro_coin_classification.ipynb \
  --ExecutePreprocessor.timeout=600
```

Окремо перший етап можна запустити командою:

```bash
python3 -m src.data_preparation
```

## Pipeline

1. Зчитування фотографій та відповідних YOLO-файлів.
2. Перетворення нормалізованих рамок у піксельні координати.
3. Перевірка анотацій, меж рамок і доступності зображень.
4. Вирізання 336 монет і масштабування кожного кропу до `64x64`.
5. Перетворення RGB-зображень у відтінки сірого.
6. Формування flattened pixels із 4096 значень.
7. Формування основної HOG-матриці розміром `(336, 1764)`.
8. Кодування класів через `LabelEncoder` і нормалізація через `StandardScaler`.
9. Стратифікований поділ: 268 навчальних і 68 тестових об'єктів.
10. Навчання десяти моделей на однакових HOG-ознаках.
11. Оцінювання accuracy, precision, recall, F1-міри та матриць помилок.
12. Фінальне порівняння точності й часу та t-SNE аналіз тестових об'єктів.

Perplexity для t-SNE автоматично обмежується розміром тестової вибірки, тому алгоритм не отримує недопустиме значення.

## Навчені моделі

- LogisticRegression
- KNeighborsClassifier
- SVC
- DecisionTreeClassifier
- RandomForestClassifier
- ExtraTreesClassifier
- GradientBoostingClassifier
- AdaBoostClassifier
- GaussianNB
- MLPClassifier

Усі моделі використовують той самий train/test split. Для алгоритмів із випадковістю встановлено `random_state=42`.

## Результати

| Модель | Тестова точність |
|---|---:|
| Логістична регресія | 0.8529 |
| Надзвичайно рандомізовані дерева | 0.7794 |
| Метод опорних векторів | 0.7647 |
| Випадковий ліс | 0.7500 |
| Багатошаровий перцептрон | 0.7500 |
| Метод k-найближчих сусідів | 0.7353 |
| Градієнтний бустинг | 0.6471 |
| Гаусівський наївний Баєс | 0.6324 |
| Дерево рішень | 0.4265 |
| AdaBoost | 0.3235 |

Найкращу accuracy показала логістична регресія — `0.8529`. Найменший час `fit` мав KNN. За правилом «найшвидша модель серед тих, що відстають від найкращої accuracy не більше ніж на 0.10» найкращий баланс точності й часу показав SVC. Точні значення часу залежать від комп'ютера і записуються у `outputs/final_model_comparison.csv` після кожного запуску.

## Висновок

HOG добре описує контури цифр, обідок і рельєф монет, тому класичні моделі змогли досягти корисної якості без CNN і великих обчислювальних ресурсів. Помилки пояснюються невеликим розміром набору, дисбалансом класів, різним освітленням, поворотами, сторонами монет і фонами. Для університетського дослідження підхід є достатньо простим, відтворюваним і наочно демонструє відмінності між багатокласовими алгоритмами.
